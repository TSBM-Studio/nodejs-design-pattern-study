# 전략 패턴
로직을 전략이라는 별도의 상호 교환 가능한 객체로 추출하여 상황에 따라 객체의 변경을 지원하는 패턴.

node 진영에서 전략 패턴은 의존성 주입 + 다형성으로 인해 거의 필수 패턴으로 쓰임.

EX: PassportStrategy, PaymentStrategy

## 예시
자세한 내용은 code 참고.

# 상태 패턴
컨텍스트의 상태에 따라 전략이 변경되는 특별한 전략 패턴.

전략 패턴은 컨텍스트가 고정이 되어있고 전략이 바뀌는 반면, 상태 패턴은 전략이 고정이 되어있고 컨텍스트가 바뀜.

## 예시
자세한 내용은 code 참고.
